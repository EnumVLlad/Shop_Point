# Views rules

- Keep Rails views focused on presentation and simple conditionals.
- Put business logic in models, policies, helpers, or interactors instead of ERB templates.
- Reuse existing layout, localization, and styling patterns before introducing new UI patterns.
- Use Rails path helpers instead of hardcoded internal URLs.
- Keep customer-facing text localizable when changing existing localized screens.
- Do not introduce API behavior through views.
